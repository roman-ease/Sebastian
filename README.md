# Sebastian — AI Work Supporter

> 有能な執事のように、あなたの業務を静かに整えます。

業務中に残した雑多なメモを、AIが日報・タスク・週報へ自動整理するローカル常駐型デスクトップアプリです。

---

## 概要

情報システム担当・社内兼務職など、**複数チャネルから突発的なタスクが飛び込む業務環境**向けに設計されています。

記録の入口は極限まで軽く。整理・清書・候補提示はセバスチャンが行います。

```
日中: グローバルショートカットでメモを随時記録
終業: 「1日を締める」で日報案を自動生成 → 確認・承認
     → タスク候補をレビューしてタスク一覧へ反映
週末: 週報案を自動生成 → 確認・承認
```

---

## 主な機能

### メモ
- デイリーメモへの自由入力（構造化不要）
- 1秒デバウンスによる自動保存
- グローバルショートカット（`Ctrl+Shift+M`）でどこからでも即起動
- **過去日・翌日ナビゲーション**（← → で日付を移動、過去日のメモも編集可能）
- 未整理メモの可視化（日報未作成時にバナー表示）

### 日報
- メモ＋タスク変更ログをもとにAIが日報案を生成
- 編集可能なドラフト → 承認で `Nippo_yyyymmdd.md` として保存
- 保存先フォルダは設定で変更可能

### タスク管理
- タスクの作成・編集・完了・保留・削除
- **開始日・終了日**の期間指定
- **進捗率**（0〜100%）の手動入力 / チェックリスト連動で自動反映
- **チェックリスト**（複数項目の追加・チェック・削除）
- **作業ノート**（タスクカードから随時追記、インライン編集）
- **変更履歴**のタイムライン表示（ステータス変更・優先度変更・日付変更など）
- **一括操作**（複数選択 → ステータス変更・アーカイブ）
- AIがメモ・日報から新規タスク候補・更新候補を提案
- 差分プレビュー（変更前 → 変更後を項目ごとに表示）
- 反映前にユーザー確認必須（自動確定なし）
- 全操作を監査ログに記録（手動操作 / AI提案を区別）

### AI × チェックリスト・進捗
- **チェックリスト自動生成**: タスクのタイトル・概要からAIが5〜7項目を提案
- **AI代理チェック**: 今日のメモとチェックリスト項目を照合し、完了済み項目を提案・確認後まとめてチェック
- **AI進捗コメント**: 進捗率・チェックリスト状況をもとにセバスチャンが執事口調で一言
- **チェックリスト連動**: チェック完了数から進捗率をワンクリックで算出・反映

### 検索
- **グローバル検索**（`/search`）でタスク・メモ・日報を横断検索
- 300msデバウンスでリアルタイム絞り込み
- 結果をタスク・メモ・日報の3セクションに分けて表示

### 週カレンダー
- 月曜始まりの週表示
- **期間タスクをスパンバー**で表示（開始日〜終了日を横断するバー、前後週への延長も表示）
- スパンバー・単日チップに進捗率・チェックリスト達成数（x/n）を表示
- 各曜日カラム下端に**タスク負荷バー**（タスク数に応じて伸縮・変色）
- カレンダー下部に切り替え可能なパネル
  - **週のサマリー**: 期日タスク合計・完了数・未完了数・高優先度数・達成率バー・週評AIコメント
  - **今週の格言**: 著名人の名言を週番号でローテーション表示

### 週報
- 週内の日報＋タスクログからAIが週報案を生成
- 承認で `Shuho_yyyymmdd.md` として保存

### 朝のブリーフィング
- アプリ起動時（1日1回）に当日期日・期限近・優先度高タスクを表示
- タスクごとの**進捗率バー**・**チェックリスト達成数**も合わせて表示

### ダッシュボード
- 今日が期日・高優先度・ピン留めタスクを一覧表示
- 各タスクに**進捗率バー**・**チェックリスト達成数**を表示
- カテゴリ別の進捗サマリー

### 終業リマインド
- 指定時刻にアプリ通知で「日報を締めませんか？」を通知（Web Notification API使用）
- 通知クリックで日報ページへ直接遷移
- 時刻・平日のみ通知を設定で変更可能
- 初回起動時にブラウザの通知許可が必要です

### データ同期
- 指定フォルダへのエクスポート／インポートによるPC間同期
- 設定画面で同期フォルダを指定してPush / Pull

## UI デザイン

クラシカル書斎をモチーフにした独自テーマを採用しています。

- **フォント**: Cinzel（見出し） / EB Garamond（本文）
- **3テーマ対応**: ライト（パーチメント）/ ダーク（夜の書斎）/ セピア（アンティーク）
  - サイドバー内のトグルでいつでも切り替え可能
- **執事イラスト**: サイドバー下部に常駐し、アプリの世界観を演出

## セットアップ

### 必要環境

- [Node.js](https://nodejs.org/) 18以上
- [Rust](https://www.rust-lang.org/tools/install)（`rustup` 経由推奨）
- [Tauri CLI prerequisites](https://tauri.app/start/prerequisites/)（Windows: WebView2 / Build Tools）

### インストール・起動

```bash
# 依存パッケージのインストール
npm install

# 開発サーバー起動（HMR有効）
npm run tauri dev

# プロダクションビルド
npm run tauri build
```

### AI設定

アプリ起動後、**設定 → AI設定** から使用するプロバイダーを選択してください。

**Gemini API（推奨）**
1. [Google AI Studio](https://aistudio.google.com/apikey) で無料APIキーを取得
2. 設定画面でAPIキーとモデル（`gemini-2.0-flash`）を入力

> ⚠️ **APIキーの取り扱いについて**
> APIキーはローカルDBに平文で保存されます。
> 共有PCや管理外端末への導入は推奨しません。

**Ollama（ローカルLLM）**
1. [Ollama](https://ollama.com) をインストール
2. モデルをプル（例: `ollama pull qwen2.5:3b`）
3. 設定画面でエンドポイントとモデル名を入力

## 設計方針

- **ローカル完結** — クラウド同期なし。データはすべて手元に。
- **AI提案は必ずユーザー確認を経由** — 自動確定処理は一切行わない。
- **AI失敗時もデータを守る** — メモ・タスクは常に手動で管理可能。
- **軽量さ優先** — 常駐しても邪魔にならない起動速度を維持。
